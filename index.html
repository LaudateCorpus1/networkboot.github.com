NetworkBoot.org

A place where network booting enthusiasts can gather and share insight and understanding.

Tools to use

  • iPXE - a scriptable network boot loader and firmware
  • wimboot - boot Windows .wim files directly using iPXE
  • PXELINUX - a flexible network boot loader

Places to discuss

  • iPXE forum
  • reboot.pro network boot forum
  • Google+ network booting community

More information

  • Installing Windows using iPXE and wimboot
  • Deploying Windows over the network using WDS
  • Etherboot/gPXE Wiki
  • Dan Kegel's page about PXE and network booting

Core technologies

  • DHCP - a protocol used to initialize network configuration for a client
  • TFTP - a protocol used to download a network boot program (NBP)
  • HTTP - a protocol used to download information from a web server
  • PXE - a way to boot Intel computers using DHCP and TFTP
  • UNDI - an API used by the PXE environment to generalize access to networking hardware

Other protocols to understand

  • iSCSI - a protocol used to access storage block devices over the network
  • AoE - a protocol used to access storage block devices over the local network only
  • NBD - a protocol used to access storage block devices over the network
  • NFS - a protocol used to access files over the network (mostly Unix)
  • SMB/CIFS - a protocol used to access files over the network (mostly Windows)
